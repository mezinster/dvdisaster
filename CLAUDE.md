## What is dvdisaster

dvdisaster provides Reed-Solomon ECC protection for optical media (CD/DVD/BD). It can create error correction data and use it to recover damaged disc images. This is an unofficial fork maintaining compatibility with upstream while adding features like CLI-only builds, GTK3 support, and cross-platform CI.

## Build Commands

```bash
# Configure (GUI version, requires GTK3 + glib2)
./configure

# Configure (CLI-only, no GTK dependency)
./configure --with-gui=no

# Build (GNU make required; use `gmake` on BSD)
make -j$(nproc)

# Useful configure flags
./configure --with-werror          # treat warnings as errors
./configure --with-debug           # debug build (-ggdb)
./configure --with-debug-printf-format  # enable printf format checking (binary not usable, build-only)

# Clean
make clean      # remove object files and binary
make distclean  # also remove configure output (Makefile.config, GNUmakefile, src/build.h)

# Verify build config
make show
```

## Running Tests

Regression tests live in `regtest/` and are bash-based, comparing output against known-good database files in `regtest/database/`.

```bash
# Run all regression tests
mkdir -p /var/tmp/regtest
./regtest/runtests.sh

# Control parallelism (default: auto-detected)
MAX_JOBS=4 ./regtest/runtests.sh

# Disable UTF-8 in test output (used in CI)
REGTEST_NO_UTF8=1 ./regtest/runtests.sh
```

Test configuration is in `regtest/config.txt` — each test can be toggled yes/no. Tests are grouped by codec: `regtest/rs01.bash`, `regtest/rs02.bash`, `regtest/rs03f.bash` (RS03 file mode), `regtest/rs03i.bash` (RS03 image mode). Common helpers are in `regtest/common.bash`.

When adding a new test: run it, then copy/modify `/dev/shm/newlog.txt` as the database reference file (see `regtest/README` for format details).

## Architecture

### Build system

- Custom `./configure` script (NOT autoconf) — implemented via `scripts/bash-based-configure`. It generates `Makefile.config` and `GNUmakefile` from `GNUmakefile.template`.
- `Makefile` only exists as a stub that tells you to run `./configure` first.
- `src/method-link.c` is auto-generated by configure — it scans `src/ecc-*.c` files and creates a `BindMethods()` function to register all codecs.

### Source organization (`src/`)

**Core/entry:**
- `dvdisaster.c` — main(), CLI argument parsing, program entry
- `dvdisaster.h` — single mega-header included by everything (no fine-grained include dependencies)
- `closure.c` — `GlobalClosure` initialization (the global state struct `Closure`)

**Codec methods (the ECC algorithms):**
- `ecc-rs01.c`, `ecc-rs02.c`, `ecc-rs03.c` — method registration entry points (each calls `register_rsXX()`)
- `rs01-*.c` — RS01 implementation (create, fix, verify, common, window)
- `rs02-*.c` — RS02 implementation (create, fix, verify, recognize, common, window)
- `rs03-*.c` — RS03 implementation (create, fix, verify, recognize, common, preferences, window)
- Each codec registers a `Method` struct with function pointers for create/fix/verify/recognize

**Reed-Solomon math:**
- `galois.c` + `galois-inlines.h` — Galois field arithmetic (GF(2^8))
- `rs-encoder-sse2.c`, `rs-encoder-altivec.c` — SIMD-optimized RS encoding

**I/O and image handling:**
- `image.c` — ISO image open/read/write abstraction
- `large-io.c` — large file I/O (>2GB support via `_FILE_OFFSET_BITS=64`)
- `udf.c` — UDF filesystem parsing for image metadata
- `scsi-layer.c` + `scsi-{linux,darwin,freebsd,netbsd,win32,simulated}.c` — platform-specific optical drive access

**Reading strategies:**
- `read-linear.c` — sequential sector reading
- `read-adaptive.c` — intelligent "divide and conquer" reading for damaged media

**GUI (GTK3, compiled out with `--with-gui=no`):**
- `main-window.c`, `menubar.c`, `preferences.c`, `welcome-window.c`
- `*-window.c` files in codec dirs — per-codec GUI panels
- `curve.c`, `spiral.c` — progress visualization widgets
- `misc-gui.c` — GUI utility functions
- Icons compiled into binary via `glib-compile-resources` from `icons/`

**Utilities:**
- `crc32.c`, `md5.c` — checksum implementations
- `bitmap.c` — sector bitmap tracking
- `crcbuf.c` — CRC buffer management for sector verification
- `random.c`, `memtrack.c`, `logfile.c`

### Key design patterns

- **Global state:** Nearly all state lives in the `GlobalClosure` struct (variable `Closure` in `closure.c`). This is accessed everywhere.
- **Method/codec plugin pattern:** Codecs register via `Method` structs with function pointers. `BindMethods()` (auto-generated) calls each `register_rsXX()`. The method list is in `Closure->methodList`.
- **GUI conditional compilation:** `WITH_GUI_YES`/`WITH_GUI_NO` preprocessor guards. When GUI is disabled, GTK types are stubbed out as void typedefs in `dvdisaster.h`.
- **NLS/i18n:** Uses gettext. The `_()` macro wraps translatable strings. `WITH_NLS_YES`/`WITH_NLS_NO` controls compilation. Locale files in `locale/`.
- **Platform abstraction:** SCSI layer uses compile-time platform selection (`SYS_LINUX`, `SYS_MINGW`, `SYS_DARWIN`, etc.).

### CI/CD

GitHub Actions workflows in `.github/workflows/`:
- `tests.yml` — regression tests on Linux, macOS (x86_64 + arm64), and Windows (MSYS2/MINGW64), for both CLI and GUI variants. Runs on PRs and pushes to master.
- `release.yml` — multi-platform release builds (macOS .dmg, Windows .zip, Linux static CLI, AppImage, .deb). Uses a `prepare-tag` job to avoid tag race conditions between parallel platform builds. Triggers on push to master/dev and on `v*` tags.
- `codeql.yml` — CodeQL static analysis on all pushes and PRs
- `stale.yml` — auto-closes stale issues with `needs-more-info` or `answered` labels

Commit message convention: lowercase prefix (`fix:`, `enh:`, `chore:`) followed by description.

### Versioning

Version is set in `configure`: `PACKAGE dvdisaster 0.79.10` with `DEFINE_INT PATCHLEVEL 6`. Build timestamp goes into `src/build.h` (auto-generated).
